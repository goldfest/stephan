# Экзаменационный проект: социальная сеть

Full-stack SPA по варианту **№5: «Социальная сеть. Размещение постов, комментарии»**.

## Что реализовано

- Архитектура **Spring Boot REST API + React SPA**.
- Аутентификация и авторизация через **JWT**:
  - регистрация;
  - вход;
  - выход на клиенте;
  - роли `USER` и `ADMIN`;
  - пользователь может изменять только свои посты/комментарии;
  - администратор может изменять/удалять любые посты/комментарии и управлять пользователями.
- Доменная модель:
  - `UserEntity`;
  - `PostEntity`;
  - `CommentEntity`;
  - `LikeEntity`;
  - `TagEntity`.
- Связи:
  - `User -> Post`: one-to-many;
  - `Post -> Comment`: one-to-many;
  - `User -> Like` и `Post -> Like`: one-to-many;
  - `Post <-> Tag`: many-to-many.
- CRUD:
  - полный CRUD для постов;
  - создание/чтение/обновление/удаление комментариев;
  - переключение лайка;
  - просмотр пользователей, смена роли и удаление пользователя для ADMIN.
- Две страницы с серверной фильтрацией:
  - **Публичная лента**: поиск по заголовку/тексту, фильтр по автору, тегу и диапазону дат;
  - **Мои посты**: поиск по заголовку/тексту, фильтр по тегу и диапазону дат.
- Фильтрация комментариев по посту на странице детального просмотра поста.
- Валидация:
  - на клиенте перед отправкой форм;
  - на сервере через DTO и Bean Validation.
- UI:
  - адаптивная верстка Bootstrap;
  - минимум три страницы;
  - навигация;
  - индикаторы загрузки;
  - сообщения об ошибках и успешных операциях.
- Хранение данных между запусками: файловая база **H2** в `backend/data/social-network-db`.
- Файл с тестовыми данными: `backend/src/main/resources/seed-data.json`.
- Swagger UI: `http://localhost:8080/swagger-ui.html`.

## Тестовые пользователи

Тестовые данные лежат в файле `backend/src/main/resources/seed-data.json` и автоматически загружаются при первом запуске, если база пустая:

| Логин | Пароль | Роль |
|---|---|---|
| `admin` | `admin123` | `ADMIN` |
| `ivan` | `user123` | `USER` |
| `anna` | `user123` | `USER` |
| `stephan` | `user123` | `USER` |

## Запуск backend

Требования: Java 21.

```bash
cd backend
./mvnw spring-boot:run
```

Backend будет доступен на `http://localhost:8080`.

Полезные ссылки:

- Swagger UI: `http://localhost:8080/swagger-ui.html`

## Запуск frontend

Требования: Node.js 20+.

```bash
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

Frontend будет доступен на `http://127.0.0.1:5173/`.

Если backend запущен на другом адресе, создай файл `frontend/.env`:

```env
VITE_API_URL=http://localhost:8080/api
```
